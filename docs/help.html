The Byrsa Archive

Introduction

This page presents the actively updated dataset that was originally created as part of my master's thesis, Burkhart 2021. The document shown in the pane on the left is an interactive version of the original thesis, while the map in the pane on the right displays the geospatial dataset contained in the actively maintained Byrsa Archive.

The archaeological features shown in this map have been digitized from maps, figures, and schematic drawings from numerous publications. The features are linked to these publications through identifier codes in the feature metadata. Sometimes, assumptions had to be made to overcome a gap in the published data: these assumptions are encoded in auxiliary features called "axes of reconstruction". In the sections below, an overview will be given of these different data types, as well as how you can navigate between them using the map's interface.

Note: the text of the thesis is identical to the original text—no corrections or alterations have been made, not even for basic typos. In contrast, the map represents a dataset that is periodically updated.

Navigation

Basic Map Controls

The map has been designed to work similarly to widely popular web mapping applications. The basic controls are as follows:

  • Zoom: scroll on the mouse wheel or double-click with the mouse.
  • Pan: click and drag, or select a feature to fly to it.
  • Select: click on a feature to select it.
  • Deselect: click on a selected feature to deselect it.

In addition, there are additional map tools provided by the following buttons:

Measure Tool: click repeatedly to measure distances between points. Press Esc twice to exit the measure tool.
Animate-On-Scroll: click to enable/disable. When enabled, scrolling through the thesis will cause the map to pan and zoom to the relevant features.
Layer Visibility: click to toggle the visibility of various layers.

Document-Map Links

In the feature info box (top right) and throughout the thesis document, all Feature ID, Figure ID, and Axis ID codes are automatically linked to their corresponding features in the map: Feature ID links to archaeological features, Figure ID links to published map extents and profile extents, Axis ID links to axes of reconstruction, and Grid ID links to the reference Grid.

These links can be used to quickly navigate through the dataset:

  • click on a link to highlight the feature it refers to.
  • Ctrl + click on a link to select the feature it refers to.

Page Numbers

All page numbers from the original thesis text have been included in the interactive document. Page breaks are indicated by ## | ##. Each ## is a clickable link to the page number in the original thesis PDF.

Archaeological Features

Archaeological features are categorized according to their function, and the category codes comprise the first two characters of the Feature ID code. The most important categories are CI (cistern), MU (wall), and PI (foundation pile). For a more comprehensive breakdown of all of the categories and subcategories, see Section 2.2.

Features are further distinguished by their ontological category, which refers to the confidence with which their presence, position, and form can be known. A general breakdown of these categories is as follows:

  • Extant features are directly attested in a previous publication in such a way that their presence, position, and form are certain. This means that one or more publications depict the feature in the way the excavators actually saw it, and in a map that can be georeferenced.
  • Reconstructed features are directly or indirectly attested in a previous publication in such a way that their presence, position, and form are probable, but not certain. This means that one or more publications either a) directly identify their depiction of the feature as a reconstruction from fragmentary finds, or b) directly describe the feature as wholly extant but does not depict it in a map that can be georeferenced, or c) never directly depict or describe the feature, but nevertheless provide enough indirect information that the feature's presence, position, and form can be inferred.
  • Hypothesized features are never directly depicted or described in a previous publication, but there is sufficient indirect evidence that their presence and position can be inferred, and that their form can be reasonably hypothesized.

Figures

Each previously published figure (map, profile, photo, etc.) is designated by a unique Figure ID, which concisely specifies which publication the figure comes from, what page number the figure is on, and who scanned the figure that was georeferenced for this project. For a full overview of the Figure ID code structure, including all pertinent citations, see Section 2.1, and especially Tables 2.1 and 2.2.

Axes of Reconstruction

For some features, published maps and descriptions were not sufficient for digitization, and so additional assumptions had to be made based on the available evidence. Many of these assumptions are encoded in "axes of reconstruction", which are lines that were drawn based on georeferenced maps or drawings from previous publications, and then extended based on additional assumptions. Each axis is designated by a unique Axis ID, which allows it to be referenced in feature metadata.
